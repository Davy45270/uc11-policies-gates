# UC11 Policies & Gates

## Expression de besoin fonctionnelle
- **Objectifs** : appliquer policies et gates d’approbation automatiques avant release.
- **Acteurs** : compliance, sécurité, QA, managers.
- **Flux clés** : définition policy → intégration pipeline → évaluation → approbation/rejet.
- **Exceptions** : dérogations documentées avec approbation explicite.
- **Critères d’acceptation** : policy‑as‑code, preuves d’approbation, traçabilité complète.

## Analyse ROI
- **Gains** : réduction des non‑conformités, accélération des validations, audit simplifié.
- **Coûts** : paramétrage règles, intégration CI/CD.
- **KPI cibles** : taux de conformité, temps moyen d’approbation, % pipelines couverts.
